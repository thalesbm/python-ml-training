🎯 Casos de uso possíveis

🔹 1. Classificação binária: prever vitória do time A
Target: team_a_win (0 = não, 1 = sim).

- Features possíveis:
    - year (ou década, como feature temporal)
    - stage_name (fase do torneio)
    - group_name
    - team_a_code, team_b_code (transformados com encoding)

Modelos indicados: Regressão logística, Random Forest, XGBoost.
- Valor didático: começar com classificação binária simples.

🔹 2. Classificação multiclasse: prever resultado (Vitória A / Vitória B / Empate)
Target: você pode criar uma coluna única a partir de team_a_win, team_b_win e draw.

- Features: mesmas do caso 1.

Modelos: Random Forest, Gradient Boosting, Redes neurais simples.
- Valor didático: introduz multiclasse, avaliação com matriz de confusão.

🔹 3. Regressão: prever o número de gols de um time
Target: team_a_score ou team_b_score.

- Features: stage_name, group_name, encoding de times.

Modelos: Regressão linear, Ridge/Lasso, Gradient Boosting Regressor.
- Valor didático: aprender métricas de regressão (RMSE, MAE).

🔹 4. Prever probabilidade de empate
Target: draw.

- Features: times, fase, ano, grupo.
Valor didático: caso de classificação altamente desbalanceado (empates são raros em mata-mata), ótimo para aprender class imbalance techniques (resampling, class weights).

🔹 5. Clustering: agrupar seleções por estilo ao longo das Copas

- Features: estatísticas agregadas (média de gols feitos/sofridos por edição).

Objetivo: descobrir padrões — ex.: times que historicamente marcam muito, mas sofrem também (Brasil anos 50), vs. defesas sólidas (Itália anos 80).

Modelos: K-Means, PCA para redução de dimensionalidade.
Valor didático: prática em aprendizado não supervisionado.
